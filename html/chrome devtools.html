chrome devtools

定义

Chrome 开发者工具（developer tools）是一套内置于 Google Chrome 中的 Web 开发和调试工具，可用来对网站进行迭代、调试和分析。[1]

下载

chrome devtools 内嵌于 chrome 浏览器，可在 google 官网下载 chrome 浏览器。

打开

可以使用以下三种方式打开 chrome devtools 。

功能

chrome devtools 有以下面板功能：

  • Elements（DOM元素）
  • Console（控制台）
  • Sources（资源）
  • Network（网络/时间线）
  • Performance（行为）
  • Memory（内存）
  • Application（应用程序）
  • Security（安全）
  • Audits（审计）

Elements

Elements 面板用于观察当前文档 DOM 的所有元素及其样式，同时可以对元素（内部文本、class 等）和样式属性进行即时修改。

Console

Console 允许开发者使用 Console API 来协助开发。开发者可以从控制台看到日志诊断消息来调试 web 应用，也可以通过 API 与 chrome devtools 进行交互。

Console Setting

点击右上角的“齿轮”，对控制台进行设置。

执行环境

可以选择其他框架或拓展程序作为执行环境，默认为 Top ，即页面的顶部。（通常也不需要修改）

过滤

可以对控制台的输出内容进行过滤。

  • 在 Filter 输入框输入文本后，与文本无法字符串匹配的输出消息将被隐藏
  • 选择右侧下拉列表，勾选不同的复选框显示与隐藏指定类型的输出

Console API

在控制台可以直接使用 javascrpt API （可以使用 shift + enter 组合键来进行换行），写完代码后按 enter 运行，其后第二三行显示“输出”和“返回值”。

输出

使用 Console API 可以在控制台进行输出，所有的输出 API 接受多个参数，输出时将多个参数连接起来，中间用空格隔开（同时也支持格式化输出）。
  • console.log|info(string)：普通消息输出
  • console.warn(string)：警告消息输出
  • console.error(string)：错误消息输出
  • console.assert(boolean, string)：若前一个判断条件为 false ，则输出后面字符串为错误消息，若为 true ，则不做任何输出
  • console.dir(element)：以对象的形式打印输出一个 DOM 元素

可视化数据结构

table() 用于可视化数据结构，可以将参数按照表格的方式显示出来。 

			console.table([{a:1, b:2, c:3}, {d:4, e:5, f:6}]);
			console.table({a: 1, b: 2, c: 3});

计时

使用 console.time(stringId) 开始计时，传入的字符串作为计时器的 ID 。
使用 console.timeEnd(stringId) 停止计时，传入的字符串作为计时器的 ID 。
停止计时时显示一共所花时间。

选择元素

  • $(selector)：选出符合选择器规则的第一个元素，相当于 querySelector()
  • $$(selector)：选出符合选择器规则的所有元素，相当于 querySelectorAll()
  • $x(XPath)：选出符合XPath规则的所有元素

事件

  • monitorEvents(element, eventType)：为元素添加指定类型事件，触发后在控制台显示事件信息
  • unmonitorEvents(element, eventType)：移除事件

清屏

可以使用 clear() 对控制台的所有消息进行清空。

Sources

Sources 面板查看页面所有资源，同时允许对 CSS 和 javascript 文件进行编辑，修改后，页面即时渲染。

利用断点功能，可以更高效地进行调试（debug）。 同时，snippet 功能可以对反复使用的代码进行保存，第二次使用时可以直接运行脚本（Ctrl + S 保存 snippet， Ctrl + Enter 运行 snippet）。

Network

Network 面板用于记录页面资源的加载情况，按照时间线（Timeline）的形式呈现出来。
打开 Network 面板后，按 F5 刷新页面后，开始捕捉页面资源加载情况。

  • 选择不同快照时，将会显示该快照当时的资源加载情况
  • 悬停 waterfall 下的方框可查看详细的加载情况

    • Performance

    Performance 面板可用于诊断页面运行过程中的行为（事件/响应/动画）。
    打开 Performance 面板后，按 Ctrl + E 开始记录，对页面进行一些操作，点击 stop 按钮停止记录。

    • FPS（Fram per second） 记录栏表示帧每秒，绿色部分越高帧率越多，用户体验也越好。顶端出现红色则表示帧率过低
    • CPU 记录栏用于记录 CPU 的占用率，可下面的 Aggregated Time 查看对 CPU 的占用分布
    • 事件记录调度栏，x 轴表示事件，y 轴表示事件栈，即顶层事件调用底层事件。

    Application

    Application 面板可以对本地存储（缓存/数据库）进行管理。

    Memory

    Memory 面板可以审查 javascript 运行过程中的内存问题。

    其他

  • chrome devtools 可以更改显示设备为移动设备
