CSS怪异现象

高度坍塌

我们知道，默认情况下，块级元素的宽度无限大，高度可由子元素撑开。 


<div style="
	background-color: #ddd;
	border: solid 1px #000;
">
	<div style="
		background-color: #ff0;
		width: 50px;
		height: 50px;
	">子元素</div>
</div>
子元素
如果此时子元素设置为浮动，子元素脱离了文档流，无法撑开父元素高度，父元素没有高度，呈现出“高度坍塌”的现象。 

<div style="
	background-color: #ddd;
	border: solid 1px #000;
">
	<div style="
		background-color: #ff0;
		width: 50px;
		height: 50px;
		float: left;
	">子元素</div>
</div>
子元素
解决方法一：使父元素创建 BFC ，在 BFC 中的块级元素可以把浮动元素包含进来，父元素的高度就可以被撑开了。开启 BFC 的方式参见 此处 。
不同开启 BFC 的方法有不同影响，此处使用的是设置 overflow 为 hidden 。 

<div style="
	background-color: #ddd;
	border: solid 1px #000;
	overflow: hidden;
">
	<div style="
		background-color: #ff0;
		width: 50px;
		height: 50px;
		float: left;
	">子元素</div>
</div>
子元素
解决方法二：在父元素内追加一个 clear : left / both 的块级元素，实现的方法有两种：
  • 直接添加该元素
  • 通过伪元素添加
两者实现原理是一样的，都是通过自身清除周围浮动元素，进而撑开父元素高度。 

<!-- 直接添加元素 -->
<div id="parent" style="
	background-color: #ddd;
	border: solid 1px #000;
">
	<div style="
		background-color: #ff0;
		width: 50px;
		height: 50px;
		float: left;
	">子元素</div>
	<div style="clear: both;"></div>
</div>
<!-- 通过伪元素添加 -->
<style>
	#parent: after {
		content: "";
		display: block;
		clear: both;
	}
</style>

上下外边距合并

根据 FC ，同一 BFC 内，兄弟块级元素的上下外边距会相互合并。 


<div style="
	overflow: auto;
	border: solid 1px #000;
">
	<div style="
		background-color: #888;
		height: 50px;
		margin-bottom: 50px;
	">下外边距 50 px</div>
	<div style="
		background-color: #888;
		height: 50px;
		margin-top: 50px;
	">上外边距 50 px</div>
</div>
下外边距 50 px
上外边距 50 px
解决方式：打破这个规则，让其中一个兄弟块级元素属于另一个 BFC 中。
此处使用的方法是让第二个兄弟元素左浮动，于是这个元素会处于另一个 BFC 中，不会与第一个兄弟元素的下外边距合并。 

<div style="
	overflow: auto;
	border: solid 1px #000;
">
	<div style="
		background-color: #888;
		height: 50px;
		margin-bottom: 50px;
	">下外边距 50 px</div>
	<div style="
		background-color: #888;
		height: 50px;
		margin-top: 50px;
		float: left;
	">上外边距 50 px</div>
</div>
下外边距 50 px
上外边距 50 px
