link

定义

link用于链接外部文档，最常使用于链接样式表。

  • link 元素是空元素，它仅包含属性。
  • 此元素只能存在于 head 部分，不过它可出现任何次数。

属性

rel

rel(Relationship)：规定当前文档与被链接文档之间的关系。

href

href(Hypertext Reference)：规定被链接文档的位置。
对href的理解参见此处。

hreflang

hreflang(Hypertext Reference Language)：规定被链接文档的语言。
语言表参见此处。

media

media：规定被链接文档将被显示在什么设备上。

  • 链接样式表时，如果有多个样式表，浏览器将会对所有样式表进行加载，然后根据media属性决定调用哪个样式表进行渲染。
  • media属性的值为一个判断条件，判断结果为真，则应用样式，判断结果为假，则不应用样式。
    • 
						<!-- 该样式表会应用在“屏幕大小不超过800px的计算机屏幕” -->
						<link rel="stylesheet" href="style.css" media="screen and (max-device-width:800px)">

    type

    type：规定被链接文档的 MIME 类型。

    sizes

    sizes：规定被链接资源的尺寸。

  • 只有当被链接资源是图标时 (rel="icon")，才能使用该属性。

    • 其他属性

    以下属性在 HTML5 中已经不支持

    • charset
    • rev
    • target

    link vs @import

    • link 是 XHTML 开始就存在的特性，除了能用于引入样式表，还能引入其他文档
    • @import 是 CSS2.1 的新特性，只能用于引入样式表
    同样用于引入样式表时，两者的下载时间不同： 
    
					<link rel="stylesheet" href="style.css">
					<style>
						@import url(style.css);
					</style>
    link 的加载和渲染参见此处
    @import 则会等待页面加载完毕后再加载，因此可能会出现 FOUC(Flash Of Unstyled Content/无样式内容闪烁) 现象：打开网页后先看到无样式的文档，过一段时间后样式才被加载进来，页面才呈现为正常内容(无样式的页面 -> 有样式的页面)。
