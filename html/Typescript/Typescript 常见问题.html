抽象类与接口有什么区别

使用上的区别

名字 能不能实例化 用途 能不能实现构造函数 其他 用例
抽象类 / abstract class 不能 只能被其他类继承（extends） 可以 类里面的方法都是抽象方法，需要在被 extends 时实现 

abstract class Foo {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
  abstract doSomething(): void;
}
接口 / interface 不能 被其他接口继承（extends），或者被类实现（implement） 不能，contructor 在 interface 中会被认为是方法 接口里面的方法需要在被 implement 实现 

interface Foo {
  name: string;
  doSomething(): void;
}

在概念上

  • 抽象类是对事物的抽象，接口是一系列行为的抽象
  • 抽象类具有继承的特点，即父子类关系。而接口仅能用于实现，相当于一个模板，承诺了能调用的方法

type 和 interface 有什么区别

虽然说 type 和 interface 在很多地方可以互相代替，但两者并不等价，表现为：

  • type 的语义为类型，即表现为变量类型
  • interface 的语义是接口，强调关系的继承和约束，即数据结构

泛型

泛型是什么

泛型允许我们去创建不同变量类型都可以使用的函数。

In languages like C# and Java, one of the main tools in the toolbox for creating reusable components is generics, that is, being able to create a component that can work over a variety of types rather than a single one. This allows users to consume these components and use their own types.
