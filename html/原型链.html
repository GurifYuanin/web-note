原型链

定义

如果说作用域链是使用变量或函数时的一种索引顺序，那么原型链就是使用对象的属性时，查找该属性的一种索引顺序。
在 javascript 中，通过函数通常有两种方法来创建对象（实例）。

  • 工厂方法
  • 构造函数 + 原型模式

工厂方法

工厂方法本质是在函数内创建一个对象，最终将该对象返回，是一种对象创建的封装。


function createObject () {
	var object = {};
	object.property = value;
	object.function = function () {};
	return object;
}
var instance = createObject(); // 调用函数创建对象

构造函数

构造函数本质依旧是函数，使用了 new 运算符创建新对象，该对象被赋予构造函数的作用域，然后执行一遍构造函数里的代码，执行过程中 this 指针指向最终被创建的新对象。 


function Obj () {
	this.property = value;
	this.function = function() {};
}
var instance = new Obj(); // 使用 new 创建新对象
使用构造函数的方法的特点，每个通过 new 创建出的对象是相互独立的，每个对象里面的属性都需要创建一遍。但有时我们需要不同实例拥有同一个属性。例如 

function Person () {
	this.say = function () {
		alert('hello world');
	};
}
var person1 = new Person();
var person2 = new Person();
person1.say();
person2.say();
alert(person1.say === person2.say); // => false
对于 person1 和 person2 ，两者执行的都是同一件事，即弹出 hello world ，但 person1 和 person2 的 say 方法各被创建了一遍，此时我们更希望它们能共同拥有同一个函数。原型模式则可以解决这个问题。

原型模式

需要留意，在 javascript 中：

  • 每个函数都有一个 prototype 属性（指针）
  • 每个对象都有一个 [[prototype]] 属性（指针）
prototype 指向一个对象（称之为原型对象），该对象初始时只拥有一个属性 constructor ， constructor 属性指回了 prototype 所在的函数。
[[prototype]] 指向和创建该对象的函数的 prototype 属性指向的原型对象。该属性无法直接访问（但在 Firefox、Safari 和 chrome 中，可以使用 __proto__ 来进行访问）。
由于每个对象都有 [[prototype]] 属性，该属性指向了一个共同的对象，因此我们可以给自定义的构造函数的 prototype 设置公共属性，之后每次创建新的实例后，创建出来的实例都能拥有相同属性。 

function Obj () {

}
Obj.prototype.say = function() {
	alert('hello world');
}
var person1 = new Obj();
var person2 = new Obj();
alert(person1.say === person2.say); // => true

原型链

从前面的图可以看到，由于每个对象都有一个 [[prototype]] 属性（Object.prototype 除外），因此自定义构造函数的原型对象（自定义构造函数.prototype）本身也有 [[prototype]] 属性，指向了 Object 函数的原型对象，Object.prototype 是一切对象（除 null 和 undefined）的根原型（这也是为什么自定义的对象能使用 toString 等方法）。当实例调用属性的时候，会先从自身对象内部开始查找，如果找不到，就到原型对象里查找，如果还是找不到，就到原型对象的 [[prototype]] （原型对象的原型对象）所指的下一个原型对象里找，以此类推。这也就是原型链。 


function Fruit () {
	this.feature = '可以吃';
}
function Apple () {
	this.feature = '圆的';
}
Apple.prototype = new Fruit(); // 可以直接将一个对象赋值给构造函数的原型对象
Apple.prototype.constructor = Apple; // 记得把原型对象的构造函数更改回来

var apple = new Apple();

alert(apple.feature); // => 圆的
alert(apple instanceof Apple); // true
alert(apple instanceof Fruit); // true
alert(apple instanceof Object); // true

实现继承

继承所有

默认情况下通过构造函数创建出来的对象的 prototype 是 Object，因此实例对象可以使用 Object 的方法。 


function Cat(name) {
	this.name = name;
}
var cat = new Cat('小白猫');
假如有个父类叫 Animal，如果我们希望 Cat 继承 Animal，使得创建出来的实例能够访问父类的原型上的方法。 

function Animal(name, gender) {
	this.name = name || '动物';
	this.gender = gender || 'female';
}
Animal.prototype.say = function() {
	console.log(this.name + '说话了！');
}

Cat.prototype = new Animal();
// 等价于这种方法
// Cat.prototype = Object.create(new Animal());
Cat.prototype.constructor = Cat;

var cat = new Cat('小白猫');
cat.say(); // => 小白猫说话了！
cat.gender; // => female
通过创建一个 Animal 实例向 Cat 的原型对象赋值，Cat 不仅会继承 Animal.prototype 上的方法，还会获得创建实例（new Animal()）时得到的属性（即 name, gender，因为 Animal 实例对象作为 Cat 的原型对象被继承，通过 Cat 构造函数创建出来的实例都共享该原型对象）。当然，因为 Cat 构造函数内也执行了 this.name = name，所以 name 属性既在 cat 实例上，也在 cat.__prototype__ 上，但 cat.name 比 cat.__prototype.name 优先获得（原型链）。
但是，由于 new Animal() 赋值给 Cat.prototype ，意味着所有通过 new Cat() 得到的实例，使用 实例.gender 访问的都是同一个属性，即 Cat.prototype.gender ，因此所有的猫都是雌性的啦。

仅继承父类 prototype 上的方法

如果只是想获得父类的原型对象，而不希望获得父类构造函数执行时生成的属性（即 name, gender）： 


// Animal 不变
Cat.prototype = Animal.prototype;

var cat = new Cat();
cat.say(); // => 小白猫说话了
cat.gender; // => undefined
cat.gender 在原型链上找不到，所以得到 undefined 。

继承构造函数代码给每个实例（可传参）

如果并不想要 Animal.prototype 上的方法和属性，只想要每个实例都能有构造函数内的 name 和 gender 属性呢？可以如此写： 


// Aniaml 不变
function Cat(name) {
	// 执行 Animal 构造函数，在这里可以传入参数
	Animal.call(this);

	// 原本 Cat 构造函数内的代码
	this.name = name;
}
var cat = new Cat();
cat.say(); // => 报错，因为 cat 实例没有从 Animal.prototype 获得方法
这个时候 cat.gender 访问的就是实例上的属性，而不是 cat.__protot__.gender 或 Cat.prototype.gender 。

new 方式继承构造函数代码给每个实例（可传参，但丢失自身原型链）

通过 new 关键字创建的实例，执行构造函数默认会将其上下文（this）对象返回，也可以显式地返回一个对象。 


function Foo(name) {
	return {
		name: name,
		say: function() {
			console.log(this.name + '说话了！')
		}
	};
}
var foo = new Foo('猫');
foo.say(); // => 猫说话了！
