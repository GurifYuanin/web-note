组件

定义

组件实质上是带有预设选项的 Vue 实例。按照之前的流程，我们通常会在 javascript 中创建 Vue 实例并传入选项参数，而使用组件，可以直接创建已经预设好的参数的 Vue 实例。 


					<script>
						// 注册组件
						Vue.component('组件名', { 选项参数 });
						// 创建实例
						new Vue({
							el: '#demo'
						});
					</script>
					<!-- 使用组件 -->
					<组件名 id="demo"></组件名>
顺序应该是先注册组件，再实例化 Vue 对象，否则会找不到组件而报错。

注册

选项

因为组件是可复用的 Vue 实例，所以它们与 new Vue 接收相同的选项[1]，个别选项除外。

  • data：数据属性
  • computed：计算属性
  • methods：方法
  • watch：侦听属性
  • template：模版
  • 生命周期钩子
  • mixins：混合
  • props：

data

数据属性，必须是个函数，使用组件的调用该函数获得数据属性的对象，以保证每个组件相互独立。 


					Vue.component('demo', {
						data: function() { // ✔
							return { ... };
						},
						data: { ... } // ✖
					});

computed & method & watch & 钩子 & mixins

与 new Vue() 相同，详见 Vue 实例 。

template

template 里的内容才是组件的本体，使用组件时，该标签将被 template 值替换。 


					<div id="demo">
						<my-message>啊啊啊我不甘心</my-message>
					</div>
					<script>
						Vue.component('my-message', {
							template: '<div>娃哈哈你又要被我替换掉了</div>'
						});
						new Vue({
							el: '#demo'
						});
					</script>

props

读取

组件内部作用域是独立的，无法直接访问 new Vue() 的选项对象的参数。 

					<div id="demo2">
						<my-demo></my-demo>
					</div>
					<script>
						Vue.component('my-demo', {
							template: '<div>{{greet}}</div>'
						});
						new Vue({
							el: '#demo2',
							data: {
								greet: 'hello world'
							}
						});
					</script>
此时就需要借助 props 属性，在组件声明该选项，然后在使用组件时为其添加对应属性。 

					<div id="demo2">
						<!-- 绑定到 vue 实例的数据属性 -->
						<my-demo v-bind:greet="greet"></my-demo>
						<!-- 直接赋值字符串 -->
						<my-demo greet="world hello"></my-demo>
					</div>
					<script>
						Vue.component('my-demo', {
							props: ['greet'],
							template: '<div>{{greet}}</div>'
						});
						new Vue({
							el: '#demo2',
							data: {
								greet: 'hello world'
							}
						});
					</script>
详细的 props 属性应该是一个对象，对象的属性值一个用于描述该属性的对象

					Vue.component('my-demo', {
						props: {
							greet: {
								type: String, // 类型
								require: true, // 是否必须
								validator: function(value) {
									return ...
								}
							}
						}
					});

修改

在组件的模版中，可以通过 props 来读取实例（new Vue()）的参数，但如果想在组件中直接修改这些参数，则会提示错误。此时修改的参数已经是组件间相互独立的，而不是来自于同一个实例的公共的数据属性。 

					<div id="demo">
						<my-button v-bind:time="time"></my-button>
						<my-button v-bind:time="time"></my-button>
					</div>
					<script>
						Vue.component('my-button', {
							template: '<input type="button" v-model="time" v-on:click="time++"/>',
							props: ['time']
						});
						new Vue({
							el: '#demo',
							data: {
								time: 0
							}
						});
					</script>
正确的做法应该是在使用组件时添加自定义事件，然后让组件的模版使用 $emit('事件名') 触发这个自定义事件。 

					<div id="demo">
						<my-button v-bind:time="time" v-on:inc="time++"></my-button>
						<my-button v-bind:time="time" v-on:inc="time++"></my-button>
					</div>
					<script>
						Vue.component('my-button', {
							template: '<input type="button" v-model="time" v-on:click="$emit(\'inc\')"/>',
							props: ['time']
						});
						new Vue({
							el: '#demo',
							data: {
								time: 0
							}
						});
					</script>
$emit() 用于触发自定义事件，传入的第一个参数为自定义事件名，其后可以传入其他形参。
  • 自定义事件如果是单条 javascript 语句，可以通过 $evnet 来获取其后的第一个参数
  • 自定义事件如果调用了一个方法（methods 属性），可以直接定义形参后使用

    • v-model

    有时我们想在组件上使用 v-model 。
  • 方法一，直接在组件标签上写 v-model="" 
    • 
					<div id="demo">
						<my-input v-model="content"></my-input>
						<div>{{content}}</div>
					</div>
					<script>
						Vue.component('my-input', {
							template: '<input type="text">'
						});
						var vm = new Vue({
							el: '#demo',
							data: {
								content: '我是默认信息'
							}
						});
					</script>
    实际上编译成为了： 
    
					<div id="demo">
						<my-input v-bind:value="content" v-on:input="content = $event"></my-input>
						<div>{{content}}</div>
					</div>
					<script>
						Vue.component('my-input', {
							template: '<input type="text">'
						});
						var vm = new Vue({
							el: '#demo',
							data: {
								content: '我是默认信息'
							}
						});
					</script>
  • 方法二，通过 props 传入数据后，在组件的 template 内写 v-model 进行绑定
    • 
					<div id="demo">
						<my-input v-bind:content="content"></my-input>
						<div>{{content}}</div>
					</div>
					<script>
						Vue.component('my-input', {
							props: ['content'],
							template: '<input type="text" v-model="content">'
						});
						var vm = new Vue({
							el: '#demo',
							data: {
								content: '我是默认信息'
							}
						});
					</script>
  • 解决方法，还是按照方法一的写法，但在写组件的 template 时候需要一些修改
    • 
					<div id="demo">
						<my-input v-model="content"></my-input>
						<div>{{content}}</div>
					</div>
					<script>
						Vue.component('my-input', {
							props: ['value'],
							template: '<input type="text" v-bind:value="value" v-on:input="$emit(\'input\', $event.target.value)">'
						});
						var vm = new Vue({
							el: '#demo',
							data: {
								content: '我是默认信息'
							}
						});
					</script>

    is

    按照之前的做法，我们可以用组件名来使用组件。 

    
					<组件名></组件名>
    同样也可以在任意标签上（HTML 标签、组件标签、 component）使用 is 属性来使用组件，其效果是相同的。 
    
					<标签 is="组件名"></标签>

    解析问题

    有些 HTML 元素，诸如 <ul>、<ol>、<table> 和 <select>，对于哪些元素可以出现在其内部是有严格限制的。而有些元素，诸如 <li>、<tr> 和 <option>，只能出现在其它某些特定的元素内部。[1] 
    
					<table id="demo">
					  	<my-row></my-row>
					</table>
					<script>
						Vue.component('my-row', {
							template: '<tr><td>hello</td><td>world</td></tr>'
						})
						new Vue({
							el: '#demo'
						});
					</script>
    此时借助 is 来解决问题。 
    
					<table id="demo">
						<tr is="my-row"></tr>
					</table>
					<script>
						Vue.component('my-row', {
							template: '<tr><td>hello</td><td>world</td></tr>'
						});
						new Vue({
							el: '#demo'
						});
					</script>
