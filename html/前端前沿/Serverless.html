Serverless

是什么

让用户不需要关注运行的各种资源（计算、存储），也不需要对其进行配置维护，只关注业务逻辑与构建，提高交付效率。

特点

访问量不确定

因为不同的服务访问量不一定相同，所以：

  • 按需加载：当有请求访问时，服务会自动启动，当一段时间服务不被使用时，服务将被关闭等待下次调用
  • 动态扩缩容：根据访问量动态调整服务实例

多主机

因为应用不在特定的某台服务器上启动，所以：

  • 状态非持久化：服务器节点不会保存应用的状态
  • 非会话保持：每次请求可能访问到不同的服务器节点，所以同一个客户端的多次请求无法保证请求到同一个服务器实例上

实现方式

由于函数本身是无状态的，无法保存状态或会话，所以应用可以被拆分为多个细粒度的函数，通过函数建的编排调度完成业务逻辑。

依赖服务

提供第三方工具，如消息队列、数据库、缓存等。

使用场景

FaaS

Function as a Serice，函数即服务。将应用分解为多个函数服务，通过多个函数之间的编排调度完成业务逻辑，应用运行所需的底层服务资源由平台统一管理。

Baas

Backend as a Service，后台即服务。通过 BaaS平台可以将数据库、文件存储和消息队列等第三方工具服务化，让用户以 API 的形式进行访问，按照使用量进行付费。
